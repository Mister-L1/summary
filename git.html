Git常用操作命令

初始化、提交

初始化一个Git仓库：git init

查看仓库的状态：git status

添加到暂存区：git add .(所有修改)

暂存区提交到仓库：git commit -m "info"(info:有意义的注释信息)

查看文件修改还未提交的部分： git diff

查看commit信息： git log (退出方法：英文状态下按q)

查看旧的版本的commit id：git reflog

退回到指定的版本:git reset --hard [commit id]

远程仓库相关命令

检出仓库：git clone https://github.com/Mister-L1/summary.git

查看远程仓库：git remote -v

添加远程仓库： git remote add [name] [url]

删除远程仓库： git remote rm [name]

修改远程仓库： git remote set-url --push[name][newUrl]

拉取远程仓库： git pull [remoteName] [localBranchName]

推送远程仓库： git push [remoteName] [localBranchName]

分支操作

查看本地分支： git branch

查看远程分支： git branch -r

查看所有分支：git branch -a (*号代表你现在所在的分支;remotes/origin/远程分支)

查看当前详细分支信息（可看到当前分支与对应的远程追踪分支）：git branch -vv

查看当前远程仓库信息：git remote -vv

创建本地分支：git branch [name] (新分支创建后不会自动切换为当前分支)

切换分支： git checkout [name]

创建分支并转到新分支:git checkout -b [name]

创建远程分支(先创建本地分支push到远程)： git push origin [name]

本地关联远程分支： git branch --set-upstream-to=origin/[远程分支名称] [本地关联分支名称]；老版本：git branch --set-upstream [本地关联分支名称] origin/[远程分支名称]

合并分支 ： 切换到要需要合并的master分支：git checkout master将master与dev合并：git merge dev(团队协作需要先拉取在合并：git pull)

删除本地分支：git branch -d [name] (-d选项只能删除已经参与了合并的分支，对于未有合并的分支是无法删除的。如果想强制删除一个分支，可以使用-D选项)

删除远程分支：git push origin --delete [name]或者git push origin :[name]

版本(tag)操作

查看版本： git tag

创建版本： git tag [v1.0]

删除版本： git tag -d [v1.0]

查看远程版本： git tag -r

创建远程版本(本地版本push到远程)： git push origin [v1.0] (push所有tag，命令格式为：git push [origin] --tags)

删除远程版本： git push origin :[v1.0]

子模块(submodule)操作

添加子模块： git submodule add [url] [path] (如： git submodule add git://github.com/soberh/ui-libs.git src/main/webapp/ui-libs)

初始化子模块： git submodule init ----只在首次检出仓库时运行一次就行

更新子模块： git submodule update ----每次更新或切换分支后都需要运行一下

删除子模块:

git rm --cached [path] 编辑“.gitmodules”文件，将子模块的相关配置节点删除掉 编辑“.git/config”文件，将子模块的相关配置节点删除掉 手动删除子模块残留的目录

关于每次提交输入账户密码问题:

原因是在添加远程库的时候使用了https的方式。。所以每次都要用https的方式push到远程库

解决:

1.查看使用的传输协议

git remote -v

2.重新设置成ssh的方式:

git remote rm origin 移出旧的http的origin git remote add origin git@github.com:Mister-L1/summary.git 添加新的git方式的origin git remote -v 再次查看协议 git push origin 再次push一下，已经不需要输入用户名密码了

Git与Svn比较

Git

优点：

个人开发：速度快、灵活，一个很大的优势就是可以不用联网；团队开发：团队中Git的地位类似与集中式版本控制系统，管理方便，逻辑明确，很容易的解决冲突,相对独立不影响协同开发;

缺点：

服务器压力太大，数据库容量暴增;不能连接到服务器上，基本上不可以工作;不适合开发人数多

Svn

优点：

管理方便，逻辑明确，符合一般人思维习惯;易于管理，集中式服务器更能保证安全性;代码一致性非常高;适合开发人数不多的项目开发

缺点：

学习周期相对而言比较长。不符合常规思维;、代码保密性差，一旦开发者把整个库克隆下来就可以完全公开所有代码和版本信息。但现在Github上面有私人仓库的功能，如果强调安全性保密性的话，花点钱还是可以实现安全需求的。

svn

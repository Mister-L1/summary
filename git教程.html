创建git文件夹:

git init

创建本地并转到分支:

git checkout -b dev

创建远程分支：

先创建本地分支 git checkout -b dev

提交到远程git push origin dev

查看分支：

git branch -a (*号代表你现在所在的分支;remotes/origin/远程分支)

git branch 本地分支

git branch -r 远程分支；

合并分支：

切换到要需要合并的分支master->git checkout master

合并->git merge dev(团队协作需要先拉取->git pull);

刪除分支:

删除本地分支->git branch -d dev

删除远程分支->git push origin --delete dev或者git push origin :dev

关于每次提交输入账户密码问题:

原因是在添加远程库的时候使用了https的方式。。所以每次都要用https的方式push到远程库
解决:

1.查看使用的传输协议

git remote -v

2.重新设置成ssh的方式:

git remote rm origin 移出旧的http的origin git remote add origin git@github.com:Mister-L1/summary.git 添加新的git方式的origin git remote -v 再次查看协议 git push origin 再次push一下，已经不需要输入用户名密码了
